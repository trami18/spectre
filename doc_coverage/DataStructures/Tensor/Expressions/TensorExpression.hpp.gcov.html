      18             : // The below values are used to separate upper indices from lower indices and
      19             : // spatial indices from spacetime indices.
      20             : //
      21             : // Tensor expressions perform as many calculations as possible in a constexpr
      22             : // context, which means working with fundamental types, specifically integer
      23             : // types, is easiest. By using sentinel values defined in one location we can
      24             : // easily control the encoding without having magic values floating around in
      25             : // many places. Furthermore, encoding all the information in the `size_t` means
      26             : // that when a failure occurs in one of the constexpr calculations it is
      27             : // reasonably easy to debug because, while encoded, the full type information is
      28             : // present. This approach can effectively be thought of as using specific bits
      29             : // in the `size_t` to mark information, using the size_t more as a bitfield than
      30             : // anything else. For human readability, we use base-10 numbers instead of
      31             : // base-2 values that would truly set individual bits.
      32             : //
      33             : // Spacetime indices are represented by values [0, `spatial_sentinel`) and
      34             : // spatial indices are represented by values
      35             : // [`spatial_sentinel`, `max_sentinel`). Lower spacetime indices are represented
      36             : // by values [0, `upper_sentinel`), and upper spacetime indices are represented
      37             : // by values [`upper_sentinel`, `spatial_sentinel`). Lower spatial indices are
      38             : // represented by values
      39             : // [`spatial_sentinel`, `spatial_sentinel` + `upper_sentinel`), and upper
      40             : // spatial indices are represented by values
      41             : // [`spatial_sentinel` + `upper_sentinel`, `max_sentinel`). Values equal to or
      42             : // above `max_sentinel` are considered invalid for representing an index.
